# MADEWELL Endless Debate Protocol v0.1
## Consensus Forge / 끝장토론 모드

목적:
현재 Debate 모드처럼 “Claude → Gemini → GPT → Red Team → Final”로 한 번씩 말하고 끝나는 구조를 금지한다.
중요 전략 안건에서는 각 모델이 서로의 주장을 반복적으로 공격/수정/재평가하며, 실제 협의점 또는 끝까지 남는 불일치 지점에 도달해야 한다.

---

# 0. 기본 원칙

끝장토론은 무한 반복이 아니다.
무한 반복은 비용과 시간만 낭비한다.

대신 아래 조건 중 하나에 도달할 때까지 반복한다.

1. 합의 도달
2. 더 이상 새 반박이 나오지 않음
3. 핵심 쟁점이 1~3개로 압축됨
4. 사용자가 지정한 최대 라운드 도달
5. 비용/시간 한계 도달

기본값:
- 일반 끝장토론: 4라운드
- 중요한 전략: 6라운드
- 전쟁회의: 8라운드
- 100-Branch와 결합 시: 5라운드 + 사전 100가설 생성

## Round 8 — Final Arbiter / 최종 판정
최종 정리자는 단순 요약하지 않는다.

🧨 폐기할 것:

# 3. 반복 조건

라운드가 끝날 때마다 아래를 평가한다.

- 새 반박이 나왔는가?
- 핵심 쟁점이 줄었는가?
- 후보안이 줄었는가?
- 점수가 바뀌었는가?
- 더 토론하면 결론이 개선될 가능성이 있는가?

아래 조건이면 한 라운드 더 반복한다.
- 새 반박이 강함
- 핵심 쟁점이 아직 3개 이상
- Red Team이 치명적 리스크 발견
- 후보안 점수 차이가 작음
- 사용자가 “더 싸워”라고 지시

아래 조건이면 종료한다.
- 상위 후보가 명확함
- 남은 쟁점이 실행 실험으로 검증 가능함
- 추가 토론보다 실행의 정보값이 큼
- 최대 라운드 도달

# 4. 100-Branch와 결합할 때

100-Branch War Game Mode와 끝장토론을 함께 사용할 때는 아래 순서로 진행한다.

1. 100개 가설/경우의 수/실패 경로 생성
2. 8~12개 카테고리로 클러스터링
3. 점수화
4. 상위 10개 decision-changing 가설 선정
5. 그 상위 10개를 놓고 끝장토론
6. 최종 실행은 1~3개 실험으로 압축

금지:
100개를 전부 길게 나열하고 끝내기.
100개 생성 자체를 결론처럼 취급하기.
